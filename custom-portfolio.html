Custom Portfolio Site

Web Design & Development / 2018

Tools Used:
HTML5, CSS3, SCSS, jQuery, Bootstrap, Jekyll, Git, Github, Cloudflare, Sketch

The Story

I needed a new, refreshed portfolio to house my latest work. Sure, I could have picked up a beautiful plug-and-play WordPress theme, paid a hosting service to setup my site with one click, and then used a plug-in to manage my content. But I decided to view this need as a learning opportunity. I wanted to know every line of code of this site. And I wanted to try out some web technologies that I'd heard a lot about but hadn't used much prior to this project: Bootstrap and Jekyll.

I chose these tools for one primary reason: quick, lightweight prototyping. I wanted to see how quickly I could put this site together without relying on heavy, burdensome features I wouldn't need. WordPress would have been overkill for my needs.

Key Design Insights

Discussion is King. Every interaction in the simulation should contribute to group discussion.

The experience should be dynamic and unpredictable, just like change is in the real world.

The simulation should be designed to be modular, so that the experience can be customized per client.

Design Artifacts

Rough sketches by one of our UX designers.

Low-fidelity wireframes used to gather user feedback and stakeholder buy-in.

High-fidelity mockups delivered to game developer.

Result

After eight months of research, design iterations, game development, and user testing, a fully-functioning Minimally Viable Product was completed and delivered to live clients during three daylong pilot sessions. Early feedback indicated a marked improvement in 'flow'-state for users.
